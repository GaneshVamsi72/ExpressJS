In web development, when a client (like a browser) wants to communicate with a server, it sends an HTTP Request.
There are different types of requests, and the most commonly used are GET and POST.

-----------------------------------------------------------------------------------------------------------------------

âœ… GET Request
A GET request is used to retrieve data from the server.

1. Data is sent in the URL (as query parameters) (String Data).

2. It is visible in the address bar.
3. It is not secure for sending sensitive data.
4. Can be cached by the browser.
5. Has length limitations (URL size limit).
6. Used for reading data, not modifying anything.

âœ… POST Request
A POST request is used to send data to the server, often to create or update something.

ğŸ”¸ Characteristics:
1. Data is sent in the body of the request (not visible in the URL) (Any Type of Data).
2. Suitable for sensitive or large data.
3. Cannot be cached.
4. Has no length limitations.
5. Mostly used for form submissions, logins, etc.
6. Used for writing data.
